# ADR-001: Гибридный форм-фактор (CLI + Web UI)

## Статус
Принято

## Контекст
Команды логистической платформы находятся на разном уровне зрелости в использовании GenAI-инструментов. Часть разработчиков готова работать с CLI, но значительная часть команд — включая системных аналитиков — работает в традиционной парадигме и предпочитает визуальные интерфейсы. Компания официально провозгласила курс на GenAI, но на практике адопция неравномерна.

Платформа должна обслуживать три типа пользователей:
- **Разработчики** — работают в терминале и IDE, предпочитают CLI
- **Системные аналитики** — предпочитают визуальные инструменты, загрузку файлов через браузер
- **CI/CD-пайплайны** — автоматизированные вызовы без участия человека

## Рассмотренные варианты

### A) CLI/SDK only
- (+) Быстрый time-to-value, проще разрабатывать и тестировать
- (+) Нативно встраивается в CI/CD и IDE
- (−) Высокий барьер входа для аналитиков и команд с низкой AI-зрелостью
- (−) Замедлит адопцию в масштабах компании

### B) Web-платформа с UI only
- (+) Низкий барьер входа для всех пользователей
- (−) Сложнее встраивать в рабочий поток разработчика (IDE, терминал)
- (−) Дольше в разработке
- (−) Неудобно для автоматизации (CI/CD)

### C) Гибрид: CLI-ядро + Web UI (выбран)
- (+) Покрывает все три типа пользователей
- (+) CLI-ядро обеспечивает автоматизацию и интеграцию
- (+) Web UI снижает барьер входа
- (−) Больше поверхность разработки и поддержки

## Решение
Гибридный подход: **CLI-ядро** (Python/Typer) является основой, **Core API** (FastAPI) предоставляет единый REST-интерфейс для всех функций генерации, **Web UI** (React) — тонкий фасад над API для визуальной работы.

Порядок реализации: CLI + API (Фаза 1) → CI/CD-плагин (Фаза 2) → Web UI (Фаза 4). Это позволяет начать приносить ценность разработчикам с первых недель.

## Последствия
- Увеличивается объём разработки (3 интерфейса вместо 1)
- Необходим чёткий контракт Core API, от которого зависят все интерфейсы
- Web UI требует дополнительной инфраструктуры (async задачи через Celery + Redis)
- Команда должна включать фронтенд-разработчика (начиная с Фазы 4)
